Project Overview

Platformer Runner Game C++ + raylib Solo Project

Project Description

This is a solo project I completed during university. The player character runs automatically, dodging enemies and jumping over obstacles to reach the finish line. The gameplay is fast-paced, designed for quick pick-up-and-play sessions and repeated challenges.

Highlights

  • Implemented multi-layered background auto-scrolling to enhance visual dynamism
  • Developed enemy AI logic including movement, collision detection, and trigger mechanisms
  • Designed and built a local leaderboard system to track player scores
  • Used plain text files for data persistence and retrieval

Retrospective

Given the five-week development timeline, I prioritized rapid prototyping to deliver core gameplay features. However, the lack of modular architecture made later feature expansion and maintenance more difficult, highlighting the importance of scalable design.

Next Steps

  • Introduce object pooling to optimize enemy spawning logic, improving performance and resource management efficiency

  • Refactor the project architecture for better modularity and maintainability

  • Add a new sliding animation for the player character to enrich movement dynamics

  • Implement level data loading from local files to support scalable stage design

(Estimated completion: December 2025)

Animal World

üêÑ Implementation of FSM and Boids Algorithm

Project Overview

Category Tech Stack Role
Algorithm Implementation C++ + raylib Solo Project

Project Description

This project was independently completed during my second year in the Game Design program, with the goal of simulating a dynamic ecosystem. Grass, sheep, wolves, and crows each follow distinct behavioral patterns:

  • Grass grows and spreads
  • Sheep search for and eat grass, avoid wolves, reproduce, or wander
  • Wolves hunt sheep, reproduce, or wander
  • Crows randomly fly in and out of the scene

Through the interaction of these behaviors, the grassland ecosystem gradually develops into a state of dynamic balance.

Highlights

  • Utilized a Finite State Machine (FSM) to manage animal behavior logic, enabling clear state transitions and responsive mechanisms
  • Applied the Boids algorithm to simulate flocking behavior of crows, enhancing realism and dynamic expression
  • Built a foundational ecological cycle system supporting grass growth and spread, animal reproduction, and resource consumption

Retrospective

The next steps include refactoring the project architecture to achieve modularity and scalability. In addition, incorporating images and animation assets will improve visual presentation and user comprehension, bringing the project closer to the standards of a commercial prototype.

Next Steps

Restructure the project architecture through modular refactoring to raise the codebase toward commercial-level quality. Add images and animations to represent different animals and plants along with their corresponding behaviors.

(Estimated completion: February 2026)

Hum to Heal

üéµ Collaborative Rhythm Game with hand-clapping, exploring the healing power of music and interaction

Project Overview

Category Tech Stack Showcase Experience
Arcade Rhythm Game Unity + C# 2023 Gotland Game Conference

Project Description

This project was completed during my first year in the Game Design program. Players stand at the four corners of the machine and clap hands with adjacent partners to hit note blocks, creating rhythmic coordination. The design encourages interaction and a sense of rhythm among players, making it enjoyable for friends to play together while also serving as an icebreaker for strangers. At the exhibition, the project received enthusiastic feedback and attracted a large number of visitors to try it out.

Role

This project was developed in collaboration with five classmates, who respectively took charge of game design and art implementation, arcade device construction, and project management, etc. I independently handled all programming and technical implementation, including note detection, rhythm synchronization, and player input logic.
